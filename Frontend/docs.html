Aspecte despre proiect

Sa se dezvolte o aplicatie Web destinata colectionarilor de recipiente (sticle, clondire, butelii, damigene, butelci,...). Pe baza facilitatilor de cautare multi-criteriala implementate, utilizatorii autentificati vor putea crea, inventaria si partaja (sub)colectii de recipiente in functie de diverse caracteristici (tip, imagine, valoare, tara, perioada de utilizare, istoric, existenta etichetei etc.), plus vor putea importa/exporta datele referitoare la recipientele detinute. Se vor genera statistici diverse ce pot fi exportate in formate deschise - minimal, CSV si PDF. Se va realiza, de asemenea, un clasament al celor mai populare recipiente, disponibil si ca flux de date RSS.

Arhitectura

Aplicatia foloseste un server GraphQL care poate fi queried pentru informatiile cerute de catre client.

Tehnologii si concepte folosite

Urmatoarele tehnologii si concepte au fost folosite:

  • HTML
  • CSS
  • JavaScript
  • TypeScript
  • GraphQL
  • Node

API

API-ul este reprezentat de un server GraphQL care asculta pe port-ul 8000 request-uri de la clienti si executa query-uri pe baza de date SQLite pentru a returna datele cerute.

Frontend

Utilizatorii logati pot

  • - crea recipiente noi
  • - crea colectii noi la care sa adauge recipientele create
  • - sterge si edita colectiile si recipientele proprii
  • - vizualiza recipiente adaugate de alti utilizatori
  • - vizualiza top al celor mai populare recipiente

Securitate

Parolele stocate in baza de date sunt encriptate folosind algoritmul SHA256

Performanta

API-ul GraphQL permite un nivel de performanta mai ridicat decat REST, intrucat clientii pot cere strict datele de care au nevoie de la server, evitandu-se fenomenele de overfetching si underfetching. Datele returnate urmaresc structura definita in query-ul trimis de client.
